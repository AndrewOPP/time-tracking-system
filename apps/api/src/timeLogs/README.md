# 🕒 Time Tracking Module

Цей модуль реалізує бекенд-архітектуру для обліку робочого часу співробітників.

### 👤 User Access (Співробітник)

**`POST /time-logs`**

- **Логіка:** Створення одиничного запису. Перевіряє доступ юзера до проєкту та сумарний ліміт годин (≤ 24) на вказану дату.
- **Сторінки Viso Time Tracking:** [ Мій тижневий трекер ]

**`POST /time-logs/bulk`**

- **Логіка:** Масове збереження (Upsert). Оновлює існуючі записи за `id` або створює нові. Розраховує ліміт годин у пам'яті перед записом для оптимізації запитів до БД.
- **Сторінки Viso Time Tracking:** [ Модалка "Затрекати години" у деталях проекту ]

**`GET /time-logs/me`**

- **Логіка:** Вибірка всіх логів поточного користувача за період `from/to` з інформацією про проєктів.
- **Сторінки Viso Time Tracking:** [ Мій тижневий трекер ]

**`PATCH /time-logs/:id`**

- **Логіка:** Редагування запису. При зміні дати або годин повторно перевіряє добовий ліміт, ігноруючи години самого запису, що редагується.
- **Сторінки Viso Time Tracking:** [ Мій тижневий трекер ]

**`DELETE /time-logs/:id`**

- **Логіка:** Видалення запису з бази даних за його ідентифікатором.
- **Сторінки Viso Time Tracking:** [ Мій тижневий трекер ]

---

### 💼 Manager Access (Менеджер / Адмін)

**`GET /time-logs/missing-report/:userId`**

- **Логіка:** Порівнює затрекані дати з робочим календарем. Автоматично ігнорує вихідні (Сб/Нд) та повертає список "порожніх" днів.
- **Сторінки Viso Time Tracking:** [ Перегляд співробітників (Для менеджера) ]

**`GET /time-logs/manager-report`** (Якщо буде потрібно шукати логи по search)

- **Логіка:** Загальний звіт по компанії. Підтримує фільтрацію за датами та пошук (`search`) за іменами працівників або назвами проєктів.
- **Сторінки Viso Time Tracking:** [ Перегляд співробітників (Для менеджера) ]

---

### ⚙️ Ключові обмеження (Constraints)

- **24h Limit:** Сума годин одного юзера за одну календарну добу не може перевищувати 24 години.
- **Transactions:** bulk операція виконуються через `Prisma.$transaction` для забезпечення цілісності даних.
- **Validation:** Кастомна валідація дат забороняє трекінг у майбутнє та обмежує діапазон звітів до 90 днів.
